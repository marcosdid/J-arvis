# F10.4 — Sandbox + Sessions + Hooks (design)

**Status:** approved 2026-05-13 (brainstorming session)
**Branch alvo:** `feat/f10.4-sandbox-sessions` (a partir de `feat/f10.3-projects-worktrees`)
**Tag final:** `f10.4-sandbox-sessions`

## 1. Goal

Port the Python sandbox + sessions + hooks subsystems to Go+Wails, with feature parity. End state: "Iniciar sessão" button on each task card spawns claude inside `ai-jail` **in the user's native terminal emulator** (per ADR-0008); the app observes the session through hook callbacks; the kanban reflects semantic status (executing / awaiting_response / idle / done / error) live; past JSONL transcripts of the worktree are listable and renderable read-only inside a side panel.

**Out of scope (deferred to F11+):** resume of a specific past claude session via `claude --resume <uuid>`. The Python `ClaudeSession` does not track claude's internal UUID and we cannot capture it (subprocess runs in an external terminal, not in a PTY we control). Each "Iniciar sessão" opens a fresh claude in the worktree; claude internally picks up context from its own JSONL files in `~/.claude/projects/<encoded-cwd>/`.

## 2. Design decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | **Scope = paridade Python.** No `--resume`, no claude-UUID tracking. Replay reads all JSONL files in `~/.claude/projects/<encoded-cwd>/` ordered by mtime. | Python doesn't support sandbox-session resume; capturing UUID is impossible without controlling claude's stdout. Drops complexity without losing user-visible behaviour. |
| 2 | **`ai-jail` runs inside a native terminal emulator** (gnome-terminal / konsole / xfce4-terminal / kitty / alacritty / foot / tilix / terminator / xterm). | ADR-0008. User types directly into claude in their own terminal. The Wails app does NOT embed a PTY. |
| 3 | **`.ai-jail` config file** is written into the worktree cwd before spawn. Contains `command = ["claude", ...optional_args]`, `rw_maps` (resolved `.git` worktree pointers), `ro_maps = []`, `hide_dotdirs = []`, `mask = []`, `allow_tcp_ports = []`. | ai-jail v0.10+ reads this file when invoked with no args. Matches Python `write_aijail_config`. |
| 4 | **Token store = in-memory map + DB mirror.** | Subprocess dies on daemon restart anyway. DB mirror gives audit/observability. |
| 5 | **Runtime = new `internal/sandbox` package, no reuse of `internal/master`.** No `creack/pty` dep — subprocess is detached (terminal emulator owns its own PTY). | Master runs claude direct in our PTY; sandbox spawns terminal-emulator-then-ai-jail externally. Different concepts; abstraction premature. |
| 6 | **Replay strategy = read ALL `.jsonl` files** in `~/.claude/projects/<encoded-cwd>/` for the worktree cwd, parse, return ordered messages grouped by file mtime. | No need to track which session produced which JSONL; the cwd uniquely identifies the conversation history claude knows about. |
| 7 | **Task → terminal cleanup = `SessionCleanupFn` injected into `TasksAPI`, same pattern as F10.3 `WorktreeCleanupFn`.** Sessions cleanup runs **before** worktree cleanup. | Subprocess writing to a removed cwd would zombie. Stop first. |
| 8 | **ai-jail availability = detect at boot + UI button disabled.** Hook server bind failure = same path (fail-soft, `sandbox_available=false`). Terminal emulator absence = same path. | Unified health surface. No surprise crashes; clear actionable state. |
| 9 | **Testing = 3 layers:** unit (fake Runtime) + integration (fake-aijail.sh shell fixture + real HTTP server) + Playwright (mock session via `JARVIS_E2E_RUNTIME=fake`). | Real ai-jail+claude in CI is slow + adds external dep. Script-jail covers the contract end-to-end. |
| 10 | **UI = card button + right-side panel (Sheet primitive from F9.11).** Transcript is read-only (no live PTY); panel shows status chip driven by hook bus emits, plus past JSONL messages. | Keeps task context; matches the reality that user types in OS terminal, not the Wails window. |

**Why `internal/hooks` is separate from `internal/sandbox`:** HTTP server / port binding / daemon lifecycle vs. subprocess + filesystem are different responsibilities.

### 4.2. `store.Session` row

```go
type Session struct {
    ID            string     `json:"id"`             // UUID v4 (ours, not claude's)
    TaskID        string     `json:"task_id"`
    WorktreeID    string     `json:"worktree_id"`    // FK; the cwd the session ran in
    HookToken     string     `json:"-"`              // mirror of in-memory; never JSON-exposed
    Status        string     `json:"status"`         // executing|awaiting_response|idle|error|done

    LastHookAt    *time.Time `json:"last_hook_at"`

```

No `ClaudeUUID` field — we don't track claude's internal session UUID.

### 4.3. `TranscriptMessage`

```go
type TranscriptMessage struct {
    Role      string    `json:"role"`        // user|assistant|tool_use|tool_result
    Content   string    `json:"content"`     // renderable text
    ToolName  *string   `json:"tool_name"`   // only when role=tool_use
    Timestamp time.Time `json:"timestamp"`
    SourceFile string   `json:"source_file"` // <uuid>.jsonl filename — lets UI group by past session

```

### 4.4. Hook server (internal, NOT on Wails surface)

```go
type Server struct {
    registry    *TokenRegistry
    bus         events.Emitter
    sessionRepo *store.SessionsRepo
    server      *http.Server
    listener    net.Listener
}

func (s *Server) Start() (port int, err error)  // listens on 127.0.0.1:0
func (s *Server) BaseURL() string                // "http://127.0.0.1:<port>"
func (s *Server) Stop() error                    // graceful shutdown
```

HTTP routes:
- `POST /api/hooks/Notification/{token}` → DB status=awaiting_response, **emit `session.status_changed` only if status actually changed** (matches Python `if prev != new` guard at router.py:79), return 204
- `POST /api/hooks/PreToolUse/{token}` → DB bump last_hook_at, bus emit `session.tool_use`, return `{"continue": true}`
- `POST /api/hooks/Stop/{token}` → DB status=idle, **emit `session.status_changed` only if status actually changed**, return 204

The `/hooks/Stop` route does NOT emit `session.stopped`. `session.stopped` is reserved for the manual `SessionsService.Stop` path (subprocess kill); claude-hook-driven Stop only flips status to `idle` (claude finished its current turn but the subprocess and worktree remain alive — claude is awaiting next user input).

Unknown token → 404. Invalid payload → 422.

## 5. Hook server lifecycle

### 5.1. Boot ordering in `main.go`

```go
ctx := context.Background()
db := store.Open(...); store.Migrate(...)

lazyBus := <existing>

tokenRegistry := hooks.NewTokenRegistry()
sessionsRepo  := store.NewSessionsRepo(db)
hookServer    := hooks.NewServer(tokenRegistry, lazyBus, sessionsRepo)

if _, err := hookServer.Start(); err != nil {
    log.Printf("hook server bind failed: %v — sandbox features disabled", err)
    // Continue boot: health.sandbox_available will be false; UI disables button.
}

terminalDetected := sandbox.DetectTerminal() // returns ("", err) if none found

sandboxAvailable := exec.LookPath("ai-jail") == nil
                     && hookServer.Started()
                     && terminalDetected != ""

runtime := sandbox.NewAijailRuntime(...)
sessionsSvc := core.NewSessionsService(
    sessionsRepo, worktreesRepo, projectsRepo,
    runtime, tokenRegistry, hookServer, lazyBus,
)
sessionsAPI := api.NewSessionsAPI(sessionsSvc)

// existing wiring …
tasksAPI := api.NewTasksAPI(tasksRepo, lazyBus, worktreesSvc.CleanupForTask, sessionsSvc.CleanupForTask)

// healthAPI gains sandbox_available from a closure capturing the boot result.
healthAPI := api.NewHealthAPI(func() bool { return sandboxAvailable })
```

**Failure modes (all fail-soft):**
- `hookServer.Start()` returns error → log, `sandbox_available=false`, daemon keeps running for non-sandbox features.
- `exec.LookPath("ai-jail")` fails → log, `sandbox_available=false`.
- `DetectTerminal()` returns no terminal → log, `sandbox_available=false`.

The three checks are **ANDed prerequisites**, not independent shutdown paths: any single failure marks the entire sandbox surface unavailable. The `hookServer` object and `sessionsAPI` are still constructed and wired regardless — they are just never invoked because `SessionsAPI.Start` guards on `SandboxAvailable()` at step 1 (§7.1) and returns `ErrSandboxUnavailable` immediately. `CleanupForTask` on a task that never started a session is a safe no-op.

### 5.2. Shutdown

Wails has no built-in shutdown hook, but `OnShutdown` callback exists. Register a hook that calls `hookServer.Stop()` (graceful `http.Server.Shutdown` with 5s timeout).

### 5.3. Token

`internal/hooks/tokens.go`:

```go
type TokenRegistry struct { mu sync.RWMutex; m map[string]string }

func NewTokenRegistry() *TokenRegistry
func (r *TokenRegistry) Generate(sessionID string) string  // crypto/rand → hex(32 bytes = 64 chars)
func (r *TokenRegistry) Resolve(token string) (sessionID string, ok bool)
func (r *TokenRegistry) Revoke(token string)
```

`crypto/rand` over `uuid.NewString` — 256 bits of entropy (64 hex chars). **Deliberate upgrade from Python**, which uses `uuid4().hex` (128 bits / 32 hex chars). Since the spec is starting from scratch and `crypto/rand` is the stdlib idiomatic choice, doubling the entropy adds zero implementation cost.

### 5.4. `.claude/settings.json` writer

`internal/sandbox/settings.go`:

```go
func WriteSettings(worktreePath, baseURL, token string) error
func RemoveSettings(worktreePath string) error
func EnsureGitignore(worktreePath string) error  // idempotent: appends ".claude/settings.json"
```

Output format (contract with claude):

```json
{
  "hooks": {
    "Notification": [{"matcher":"*","hooks":[{"type":"command","command":"curl -sS -X POST 'http://127.0.0.1:PORT/api/hooks/Notification/TOKEN' --data-binary @-"}]}],
    "PreToolUse":   [{"matcher":"*","hooks":[{"type":"command","command":"curl -sS -X POST 'http://127.0.0.1:PORT/api/hooks/PreToolUse/TOKEN' --data-binary @-; exit 0"}]}],
    "Stop":         [{"matcher":"*","hooks":[{"type":"command","command":"curl -sS -X POST 'http://127.0.0.1:PORT/api/hooks/Stop/TOKEN' --data-binary @-"}]}]
  }
}
```

The `; exit 0` in PreToolUse is critical: claude blocks on non-zero exit from a PreToolUse hook. We never want to block a tool call due to our hook server.

### 5.5. Security model

- Hook listener bound to `127.0.0.1` only (kernel rejects external).
- Token-per-session (256 bits crypto/rand) restricts blast radius even if another local process discovers the port.
- Token revoked on Stop; subsequent hooks for that token receive 404.
- Settings file `.claude/settings.json` is gitignored.
- `.ai-jail` config file lives only in the worktree (J-arvis-managed cwd; not pushed).

### 6.2. `.ai-jail` config writer

`internal/sandbox/aijail_config.go`:

```go
// WriteAijailConfig creates <cwd>/.ai-jail. Overwrites any prior file.
// `claudeArgs` will become `["claude", ...claudeArgs]` as the command;
// callers in F10.4 pass `nil` (default profile = bare `claude`).
// Catalog/permission-profile-driven args ship in F10.5.
//
// `rwMaps` is populated by walking `cwd` and resolving any `.git` files
// (gitlinks) to their original `.git/worktrees/<name>/../..` directories.
// Without these mounts, worktrees inside the jail report
// "fatal: not a git repository".
func WriteAijailConfig(cwd string, claudeArgs []string) error

// discoverGitDirs walks cwd and its immediate children, resolving each
// `.git` file (gitlink) to the originating repo root.
func discoverGitDirs(cwd string) []string

func RemoveAijailConfig(cwd string) error
```

File format (TOML-like, written verbatim — ai-jail's parser is fixed). Two cases:

**With at least one resolved git dir (one or more `rw_maps` entries):**
```
command = ["claude"]
rw_maps = [
    "/home/user/project/.git",
]
ro_maps = []
hide_dotdirs = []
mask = []
allow_tcp_ports = []
```

**With zero resolved git dirs (empty `rw_maps`):**
```
command = ["claude"]
rw_maps = []
ro_maps = []
hide_dotdirs = []
mask = []
allow_tcp_ports = []
```

`command` is always `json.Marshal` of `append([]string{"claude"}, claudeArgs...)`. In F10.4 `claudeArgs` is always `nil`, producing `command = ["claude"]`. F10.5 wires per-task `claudeArgs` from the catalog.

### 6.3. Runtime

`internal/sandbox/runtime.go`:

```go
type RuntimeSpec struct {
    Cwd      string   // worktree path (for monorepo) or worktree[0].parent (multi-repo)
    Terminal string   // resolved from DetectTerminal()
}

type Handle struct {
    PID int
    // No PTY handle — subprocess is detached, owned by the terminal emulator.
}

type Runtime interface {
    Spawn(ctx context.Context, spec RuntimeSpec) (Handle, error)
    Kill(ctx context.Context, h Handle) error
}

func NewAijailRuntime() Runtime
```

`AijailRuntime.Spawn` (production):
1. Build inner cmd: `["ai-jail"]` (no args; reads `.ai-jail` config in cwd).
2. Build full argv: `BuildTerminalCommand(spec.Terminal, inner)`.
3. `cmd := exec.CommandContext(ctx, argv[0], argv[1:]...); cmd.Dir = spec.Cwd; cmd.Start()`.
4. Return `Handle{PID: cmd.Process.Pid}`. Subprocess detached; caller doesn't `Wait`.

`AijailRuntime.Kill`: `syscall.Kill(h.PID, SIGTERM)`. Idempotent — `ESRCH` (no such process) is success.

## 7. Session lifecycle

### 7.1. `Start(ctx, taskID)`

```
1. Guard:
   - Load task; if core.IsTerminal(task.state) → ErrTaskInTerminalState (422)
   - Count active sessions for taskID (status NOT IN done/error); if > 0 → ErrTaskAlreadyHasActiveSession (409)
   - SandboxAvailable() check → ErrSandboxUnavailable (503)
2. Resolve worktrees + cwd:
   - List worktrees of task via WorktreesRepo.ListByTask.
   - If empty → call core.WorktreesService.CreateForTask (NEW helper; ports Python `_create_worktrees_atomic`).
   - Derive cwd:
       len(worktrees) == 1 → cwd = worktrees[0].Path  (monorepo)
       len(worktrees) >  1 → cwd = filepath.Dir(worktrees[0].Path)  (multi-repo parent)
     Note: in the multi-repo case all worktrees share the same parent dir by construction
     (Python `_create_worktrees_atomic` enforces this), so `worktrees[0]` is arbitrary — no
     sort or stable-ordering required.
3. Token: token := tokenRegistry.Generate(sessionID).
4. Settings: sandbox.WriteSettings(cwd, hookServer.BaseURL(), token); sandbox.EnsureGitignore(cwd).
5. AiJail config: sandbox.WriteAijailConfig(cwd, nil).
6. Terminal: terminal, err := sandbox.DetectTerminal(); on error → ErrSandboxUnavailable (503).
7. Spawn: handle, err := runtime.Spawn(ctx, RuntimeSpec{Cwd: cwd, Terminal: terminal})
8. Persist: store.SessionsRepo.Insert(Session{
       ID, TaskID, WorktreeID=worktrees[0].ID,
       HookToken=token, Status=executing, PID=&handle.PID,
       StartedAt=now,
   }).
9. bus.Emit("session.started", session).
10. Return *Session.

Rollback policy (in reverse order, best-effort, errors logged):
- Step 8 (Persist) fail → kill subprocess from step 7 + revoke token from step 3 + remove files from steps 4–5.
- Step 7 (Spawn) fail → no subprocess to kill; revoke token from step 3 + remove files from steps 4–5.
- Step 5 (`.ai-jail` writer) fail → revoke token from step 3 + remove `.claude/settings.json` from step 4.
- Step 4 (settings writer) fail → revoke token from step 3.
- Step 3 (token generate) fail → no state created; nothing to roll back.
- Worktrees from step 2 are NEVER rolled back — they are idempotent FS state usable on retry.
```

### 7.2. `Stop(ctx, sessionID)`

```
1. Load row via store.SessionsRepo.GetByID; 404 if missing.
2. Idempotency: if row.Status IN (done, error) → return nil (no-op).
3. runtime.Kill(ctx, Handle{PID: *row.PID}) — sends SIGTERM. Tolerant: ESRCH = OK.
4. tokenRegistry.Revoke(row.HookToken).
5. Derive cwd: load worktree via store.WorktreesRepo.GetByID(row.WorktreeID); cwd =
   worktree.Path for monorepo (one worktree linked) or filepath.Dir(worktree.Path) for multi-repo.
   Use store.WorktreesRepo.ListByTask(row.TaskID) length to discriminate.
6. sandbox.RemoveSettings(cwd) — best effort; failure logged not propagated.
7. sandbox.RemoveAijailConfig(cwd) — best effort.
8. store.SessionsRepo.UpdateStatus(sid, status=done, ended_at=now). On runtime.Kill error: status=error.
9. bus.Emit("session.stopped", session).
```

If step 3 fails (e.g., process unreachable for non-ESRCH reason), status → `error`. UI shows error chip; idempotent retry is no-op since terminal.

### 7.3. `CleanupForTask(ctx, taskID)` — invoked by `TasksAPI` cleanup hook

```
1. activeSessions := store.SessionsRepo.ListActiveByTask(taskID)
2. for each session: Stop(ctx, session.ID) — tolerant: failures logged.
3. Return nil even if some Stops failed (best-effort; aligned with F10.3 cleanup semantics).
```

### 7.4. `GetTranscript(ctx, sessionID)`

```
1. Load session row.
2. Resolve cwd same way as §7.1 step 2 (worktrees → cwd).
3. encodedCwd := strings.ReplaceAll(filepath.Clean(cwd), string(filepath.Separator), "-")
4. dir := filepath.Join(claudeHome(), "projects", encodedCwd)
5. files, err := sandbox.FindTranscriptFiles(dir); on err or empty → return [] (graceful)
6. Concatenate all files' parsed messages, ordered by file mtime ASC, then message timestamp ASC within file.
7. Tag each message with SourceFile = filepath.Base(jsonl path).
8. Return.
```

`claudeHome()` resolution order: `$JARVIS_CLAUDE_HOME` (test override), `$HOME/.claude`, fail.

Note: this returns transcripts of ALL sessions claude has ever run in this worktree — not only the current `sessionID`. This is intentional: claude doesn't track per-our-session-row mapping and our model is "the worktree is the conversation context". UI groups messages by `source_file` for readability.

## 8. `TasksAPI` cleanup wiring (cross-cutting)

```go
type SessionCleanupFn func(ctx context.Context, taskID string) error

func NewTasksAPI(
    repo TasksRepo, bus events.Emitter,
    worktreeCleanup WorktreeCleanupFn,
    sessionCleanup SessionCleanupFn,   // NEW
) *TasksAPI

// Order in Patch (when state becomes terminal) AND in Discard:
//   1. emit task.updated / task.discarded
//   2. a.runSessionCleanup(ctx, id)     // NEW — stop subprocesses first
//   3. a.runWorktreeCleanup(ctx, id)    // F10.3 — remove FS after subprocess gone
```

**Why session cleanup before worktree:** if worktree disappears first, the subprocess of claude that's still writing to it becomes a zombie writing to a non-existent path. Stop first guarantees clean shutdown.

`runSessionCleanup`: same nil-tolerant pattern as `runWorktreeCleanup`. Errors logged via `log.Printf`, never propagated.

## 9. Event taxonomy

| Event | Trigger | Payload shape |
|---|---|---|
| `session.started` | `Start` ok | `*store.Session` |
| `session.stopped` | `SessionsService.Stop` only (manual subprocess kill) | `{id, task_id}` |
| `session.status_changed` | `/hooks/Notification` or `/hooks/Stop`, **only when status actually changes** | `{id, task_id, previous, current}` |
| `session.tool_use` | `/hooks/PreToolUse` | `{id, task_id, tool}` |

Semantic split (matches Python):
- `session.stopped` = "we killed the subprocess" (terminal state `done` or `error`). Single-source from the service path; never fires from hook routes.
- `session.status_changed` = "claude reported a state transition" (executing ↔ awaiting_response ↔ idle). UI distinguishes "still alive but idle" from "killed" via this split.

No `session.exited` event: we don't watch the subprocess (no PTY, no `Wait` goroutine). Crash detection happens via the user observing the terminal emulator window OR via a future timeout-without-hook-ping heuristic (out of scope for F10.4).

## 10. Transcript parsing

### 10.1. File location

Claude writes to `~/.claude/projects/<encoded-cwd>/<session-uuid>.jsonl`, where `<encoded-cwd>` is the absolute cwd with `/` replaced by `-`. Example: cwd `/home/x/proj--feat` → `-home-x-proj--feat`.

```go
// FindTranscriptFiles returns absolute paths of all .jsonl files in dir,
// ordered by file mtime ascending. Returns nil, nil if dir doesn't exist.
func FindTranscriptFiles(dir string) ([]string, error)

// ParseTranscript reads one JSONL file. Tolerant: skips malformed lines
// (logs but doesn't fail). Future claude schema additions are ignored.
func ParseTranscript(path string) ([]TranscriptMessage, error)
```

### 10.2. Parsing

Tolerant line-by-line scan:

```go
type rawEntry struct {
    Type      string          `json:"type"`
    Message   json.RawMessage `json:"message"`
    Content   json.RawMessage `json:"content"`
    Timestamp time.Time       `json:"timestamp"`
}

// Mapping rules:
//   type=user      → 1 TranscriptMessage{role=user, content=plain text}
//   type=assistant → fan-out: 1 per content block
//                    (text → role=assistant; tool_use → role=tool_use + ToolName)
//   type=tool_result → 1 message{role=tool_result, content=stringified}
//   unknown type → skip + log (non-fatal)
//   malformed JSON line → skip + log (non-fatal)
```

Schema drift mitigation: testdata fixtures captured from real claude (versioned in `internal/sandbox/testdata/transcripts/`). Parser failures on a fixture trigger investigation, not test breakage cascade.

### 10.3. Verification of encoding

A single integration-test fixture confirms the `/` → `-` encoding matches what claude actually produces. If a future claude version changes the encoding scheme, only the fixture (and `FindTranscriptFiles` directory selection) needs updating.

### 12.2. Integration (`//go:build integration`)

The fake-aijail.sh contract is precisely defined here so the implementer has no ambiguity:

```bash
#!/usr/bin/env bash
# testdata/fake-aijail.sh
# Sleeps until terminated by SIGTERM (so tests can verify the Kill path).
# Does NOT call curl — hook fixtures are driven externally by the test
# harness against the real hook server.
# The echo is for human debugging when running the script outside a test;
# tests do NOT assert on stdout (the subprocess is detached/terminal-launched,
# stdout goes to the terminal emulator window, not a pipe).
echo "fake-aijail: spawned in $(pwd)" >&2
trap 'exit 0' TERM INT
sleep 600 &
wait $!
```

`#!/usr/bin/env bash` (not `/bin/sh`) — needed so `trap … TERM` + `wait` work reliably across dash/ash variants. The `sleep 600 &; wait $!` pattern (instead of busy `while :; do sleep 1; done`) ensures SIGTERM is delivered immediately, not buffered until the next sleep iteration.

Tests:

- `internal/sandbox/integration_test.go`: spawns fake-aijail.sh via `AijailRuntime` (overriding the binary name to point at testdata). Verifies Spawn returns PID, sleeps, Kill terminates it (process gone).
- `internal/hooks/integration_test.go`: real server on `127.0.0.1:0`. Test sends real HTTP POST. Validates token resolution + bus emit shape end-to-end.
- `internal/core/sessions_integration_test.go`: combines fake-aijail (via `AijailRuntime` with overridden binary path) + real hook server + real DB. After `Start`, test makes real HTTP POST to `<hookServer.BaseURL()>/api/hooks/Notification/<token>` (token resolved via registry); validates status transition in DB + bus emit. After `Stop`, validates subprocess dead + token revoked. **This is the most valuable test — it asserts the contract that holds the subsystem together.**

### 12.3. E2E (Playwright)

`ui/e2e/sessions-flow.spec.ts` — 4 new tests:

1. Card shows enabled `▶ Iniciar sessão` when sandbox available; disabled when not (env `JARVIS_E2E_NO_SANDBOX=1` makes `cmd/jarvis-e2e-http` set sandbox_available=false).
2. Click creates session; panel opens; chip shows `executing`.
3. Simulated hook (POST `/e2e/sessions/simulate-hook`) drives the UI live: chip + transcript update without reload.
4. Stop closes the session; chip becomes `done`; transcript remains visible; row moves to history group.

New endpoint in `cmd/jarvis-e2e-http`: `POST /e2e/sessions/simulate-hook` accepts `{session_id, event, payload}` and replays it against the actual hook server. Lets Playwright drive the full chain without needing claude+ai-jail real on CI.

Boot toggle: env `JARVIS_E2E_RUNTIME=fake` makes `cmd/jarvis-e2e-http` inject a `fakeRuntime` instead of `AijailRuntime`. The fakeRuntime's `Spawn` returns a fixed PID without forking; `Kill` is no-op. Required because we don't want CI to need `ai-jail` installed.

## 13. Risk register

| # | Risk | Mitigation |
|---|------|------------|
| 1 | Claude JSONL schema changes between versions | Tolerant parser (skip unknown fields & types); versioned fixtures in testdata; replay degrades to "no transcript" gracefully, never crashes. |
| 2 | Hook server port leaks to another local process | Bind `127.0.0.1` only + token-per-session (256 bits crypto/rand) + revoke on Stop. Accepted (same model as Python). |
| 3 | Race between subprocess exit and manual Stop | DB row status is source of truth; idempotent: Stop of already-terminal session is no-op. `bus.Emit("session.stopped")` fired at most once via atomic check-and-set in service. |
| 4 | `ai-jail` crashes mid-spawn (settings written, subprocess crashes immediately) | Subprocess detached via terminal emulator; we only know PID. If terminal closes immediately, our row stays `status=executing` until the user opens SessionPanel and clicks Stop. **Daemon restart does NOT purge stale `executing` rows** (in-memory registry rebuilds empty; rows are not auto-marked terminal). Accepted UX trade-off in F10.4; F10.5+ `last_hook_at`-driven stale-row heuristic addresses this. |
| 5 | Session cleanup blocks worktree cleanup if subprocess hangs | `runtime.Kill` is SIGTERM only (idempotent via ESRCH). No wait/timeout — fire and forget. `CleanupForTask` iterates tolerant. |
| 6 | Hook server fails to bind (port unavailable) | Boot fail-soft: log + `sandbox_available=false`. UI button disabled with reason. Daemon still runs for non-sandbox features. |
| 7 | Settings.json overwrites user's custom claude settings | The `.claude/` dir lives inside the worktree (per-task, isolated). Worktree is J-arvis-managed throwaway state. No user content collision. |
| 8 | Hook PreToolUse returns non-zero, claude blocks | `; exit 0` suffix in command guarantees we never block claude's tool execution regardless of curl/server state. |
| 9 | Token leak via logs (DB row mirror, error messages) | `HookToken` field has `json:"-"` tag; logging code reviewed to never log full `Session` struct unredacted. |
| 10 | `.ai-jail` config file rejected by ai-jail (version mismatch) | Spawn fails fast with stderr from terminal emulator. Caller sees runtime error; Start rolls back. UI shows toast. |
| 11 | Terminal emulator forks immediately, ESRCH on Kill | Captured: Kill treats ESRCH as success (terminal closed, ai-jail exited naturally). Status → done. |
| 12 | Multi-repo cwd derivation: `worktree[0].parent` assumes Python's create-worktrees structure | Add unit test for both monorepo + multi-repo cwd derivation. Document the assumption explicitly in code. |

## 15. What this design does NOT include (out of scope, future)

- **Resume of a specific past claude session** (`claude --resume <uuid>`): no UUID tracking in F10.4. Each start opens fresh; claude continues from its own JSONL accumulated for the cwd.
- **Live PTY in the Wails app** (xterm.js): user types in the OS terminal emulator launched by `ai-jail`.
- **`session.exited` event** (subprocess unexpected exit): no `Wait` goroutine; subprocess is detached and owned by the terminal emulator. F10.5+ could add stale-row heuristic via `last_hook_at`.
- **Master session integration** (F10.5).
- **MCP tools driven by claude inside ai-jail** (F10.5).
- **Catalog / permission profiles applied to spawn args** (F10.5). F10.4 always writes `command = ["claude"]`.
- **Notifications/sink** (libnotify) wired to status changes (F10.7 OS integration).
- **Run-from-panel Docker integration** (F10.6).
