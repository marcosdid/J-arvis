# F10.4 — Sandbox + Sessions + Hooks (design)

**Status:** approved 2026-05-13 (brainstorming session)
**Branch alvo:** `feat/f10.4-sandbox-sessions` (a partir de `feat/f10.3-projects-worktrees`)
**Tag final:** `f10.4-sandbox-sessions`

## 1. Goal

Port the Python sandbox + sessions + hooks subsystems to Go+Wails, with full feature parity including resume + replay. End state: "Iniciar sessão" button on each task card spawns claude inside `ai-jail` with hook-driven semantic status reaching the kanban live, plus past sessions are listable per task and resumable via `claude --resume`.

## 2. Design decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | **Scope = full F10.4 including resume + replay.** | Provides complete dogfood loop. Resume is essentially free (just `claude --resume <uuid>`); replay reads claude's own JSONL files — no new schema. |
| 2 | **Token store = in-memory map + DB mirror (paridade Python).** | Subprocess dies on daemon restart anyway; persistent token serves no recovery purpose. DB mirror gives audit/observability. |
| 3 | **Runtime = new `internal/sandbox` package, no reuse of `internal/master`.** | Master runs claude direct (1 singleton, no per-task cwd); sandbox runs ai-jail+claude (N per task, with cwd+settings injection). Different concepts; abstraction premature. |
| 4 | **Resume strategy = `claude --resume <uuid>` + read claude's JSONL for replay.** | No new schema. Schema drift is the risk; mitigation is a tolerant parser + versioned testdata fixtures. |
| 5 | **Task → terminal cleanup = `SessionCleanupFn` injected into `TasksAPI`, same pattern as F10.3 `WorktreeCleanupFn`.** | Familiar code path; runs *before* worktree cleanup so subprocess shuts down cleanly before its cwd is removed. |
| 6 | **ai-jail availability = detect at boot + UI button disabled with tooltip.** | Health-driven UX surfacing; failure mode well-understood; no surprise mid-action crash. |
| 7 | **Testing = 3 layers: unit (fake runtime) + integration (shell-script fake jail) + Playwright (mocked sessions endpoint).** | Real ai-jail in CI is slow + adds external dep; script-jail covers the contract end-to-end without those costs. |
| 8 | **UI = card button + right-side panel (Sheet primitive from F9.11).** | Keeps task context; transcript needs vertical real estate; reuses primitive. |

**Why `internal/hooks` is separate from `internal/sandbox`:** HTTP server / port binding / daemon lifecycle vs. subprocess + filesystem are different responsibilities. The UI could theoretically replace the hook transport (e.g., Unix socket) without touching the sandbox layer.

**Why `core/sessions` is the only package that knows all four subsystems:** centralizes orchestration; sandbox/hooks/store stay pure and independently testable.

## 4. API surface

### 4.1. `SessionsAPI` (Wails-bound)

```go
type SessionsAPI struct { svc SessionsService }

func (a *SessionsAPI) Start(taskID string) (*store.Session, error)
func (a *SessionsAPI) Stop(sessionID string) error

func (a *SessionsAPI) GetTranscript(sessionID string) ([]TranscriptMessage, error)
```

### 4.2. `store.Session` row

```go
type Session struct {
    ID            string     `json:"id"`             // UUID v4
    TaskID        string     `json:"task_id"`
    WorktreeID    string     `json:"worktree_id"`    // FK
    ClaudeUUID    *string    `json:"claude_uuid"`    // discovered from stdout
    HookToken     string     `json:"-"`              // mirror; NEVER JSON-exposed
    Status        string     `json:"status"`         // executing|awaiting_response|idle|error|done
    PID           *int       `json:"pid"`
    StartedAt     time.Time  `json:"started_at"`
    EndedAt       *time.Time `json:"ended_at"`
    LastHookAt    *time.Time `json:"last_hook_at"`
}
```

### 4.3. `TranscriptMessage`

```go
type TranscriptMessage struct {
    Role      string    `json:"role"`        // user|assistant|tool_use|tool_result
    Content   string    `json:"content"`     // renderable text
    ToolName  *string   `json:"tool_name"`   // only when role=tool_use
    Timestamp time.Time `json:"timestamp"`
}
```

### 4.4. Hook server (internal, NOT on Wails surface)

```go
type Server struct { registry *TokenRegistry; bus events.Emitter; sessionRepo *store.SessionsRepo }

func (s *Server) Start() (port int, err error)  // listens on 127.0.0.1:0
func (s *Server) BaseURL() string                // "http://127.0.0.1:<port>"
func (s *Server) Stop()
```

HTTP routes:
- `POST /api/hooks/Notification/{token}` → DB status=awaiting_response, bus emit `session.status_changed`, return 204
- `POST /api/hooks/PreToolUse/{token}` → DB bump last_hook_at, bus emit `session.tool_use`, return `{"continue": true}`
- `POST /api/hooks/Stop/{token}` → DB status=idle, bus emit `session.stopped`, return 204

Unknown token → 404. Invalid payload → 422.

## 5. Hook server + token + settings.json

### 5.1. Token

`internal/hooks/tokens.go`:

```go
type TokenRegistry struct { mu sync.RWMutex; m map[string]string }

func (r *TokenRegistry) Generate(sessionID string) string  // crypto/rand → hex(32 bytes = 64 chars)
func (r *TokenRegistry) Resolve(token string) (sessionID string, ok bool)
func (r *TokenRegistry) Revoke(token string)
```

`crypto/rand` over `uuid.NewString` — 256 bits of entropy for hook auth.

### 5.2. Settings writer

`internal/sandbox/settings.go`:

```go
func WriteSettings(worktreePath, baseURL, token string) error
func RemoveSettings(worktreePath string) error
func EnsureGitignore(worktreePath string) error  // idempotent: appends ".claude/settings.json"
```

Output format (identical to Python — contract with claude):

```json
{
  "hooks": {
    "Notification": [{"matcher":"*","hooks":[{"type":"command","command":"curl -sS -X POST 'http://127.0.0.1:PORT/api/hooks/Notification/TOKEN' --data-binary @-"}]}],
    "PreToolUse":   [{"matcher":"*","hooks":[{"type":"command","command":"curl -sS -X POST 'http://127.0.0.1:PORT/api/hooks/PreToolUse/TOKEN' --data-binary @-; exit 0"}]}],
    "Stop":         [{"matcher":"*","hooks":[{"type":"command","command":"curl -sS -X POST 'http://127.0.0.1:PORT/api/hooks/Stop/TOKEN' --data-binary @-"}]}]
  }
}
```

The `; exit 0` in PreToolUse is critical: claude blocks on non-zero exit from a PreToolUse hook. We never want to block a tool call due to our hook server.

### 5.3. Security model

- Hook listener bound to `127.0.0.1` only (kernel rejects external).
- Token-per-session (256 bits crypto/rand) restricts blast radius even if another local process discovers the port.
- Token revoked on Stop; subsequent hooks for that token receive 404.
- Settings file `.claude/settings.json` is gitignored (writer adds entry).

## 6. Session lifecycle

### 6.1. `Start(ctx, taskID)`

```
1. Guard:
   - Load task; if core.IsTerminal(task.state) → ErrTaskInTerminalState (422)
   - Count active sessions for taskID (status NOT IN done/error); if > 0 → ErrTaskAlreadyHasActiveSession (409)
2. Worktree resolve:
   - List worktrees of task (via store.WorktreesRepo.ListByTask)
   - If none → derive cwd from `<project.path>/../<project.name>--<slug(task.branch)>` and
     call WorktreesService.CreateForTask (NEW helper in WorktreesService, reuses git add).
   - Pick worktree[0] for spawn cwd.
3. Token: token := tokenRegistry.Generate(sessionID); registered in-memory.
4. Settings: sandbox.WriteSettings(worktreePath, hookServer.BaseURL(), token); ensure gitignore.
5. Runtime: runtime.Start(ctx, RuntimeSpec{
       AijailBin: "ai-jail", Cwd: worktreePath,
       Args: []string{"run", "--", "claude"},
       Env: os.Environ(),
   }) → *Handle (pid, pty fd).
6. Persist: store.SessionsRepo.Insert(Session{id, task_id, worktree_id, hook_token, status=executing, pid, started_at=now}).
7. Reader goroutine:
   - Scans PTY output line-by-line.
   - regexp `Session ID: ([0-9a-f-]{36})` first match → store.UpdateClaudeUUID(sid, uuid) + emit session.identified.
   - On EOF/exit → store.UpdateStatus(sid, done, ended_at=now) + emit session.exited.
8. bus.Emit("session.started", session).
9. Return *Session.

Rollback policy:
- Any failure in steps 3-6 reverses in reverse order.
- Token: tokenRegistry.Revoke(token).
- Settings: sandbox.RemoveSettings(worktreePath) (best-effort; log if fails).
- Runtime: runtime.Stop(handle) (only if started).
- DB row: deleted only if persisted (step 6).
- Worktree NEVER rolled back (idempotent — retry should not lose FS state).
```

### 6.2. `Stop(ctx, sessionID)`

```
1. Load row; 404 if missing.
2. Idempotency: if row.Status IN (done, error) → return nil (no-op).
3. runtime.Stop(handle): SIGTERM, wait 5s, SIGKILL on timeout.
4. tokenRegistry.Revoke(token).
5. sandbox.RemoveSettings(worktreePath) — best effort; failure logged not propagated.
6. store.UpdateStatus(sid, done, ended_at=now).
7. bus.Emit("session.stopped", session).
```

If step 3 fails (e.g., subprocess unreachable), status → `error` instead of `done`. UI shows error chip; user can retry Stop (no-op since row terminal) or manual cleanup.

### 6.3. `Resume(ctx, sessionID)`

```
1. Load original row.
2. Require row.ClaudeUUID != nil → else ErrClaudeUUIDNotCaptured (422).
3. Same as Start steps 3-9, but Args: []string{"run", "--", "claude", "--resume", *row.ClaudeUUID}.
4. Persist NEW session row (different ID; same TaskID/WorktreeID; status=executing).
5. Old row stays as historical (no modification).
```

### 6.4. `CleanupForTask(ctx, taskID)` — invoked by `TasksAPI` cleanup hook

```
1. activeSessions := store.ListByTask(taskID, status NOT IN done/error)
2. for each session: Stop(ctx, session.ID) — tolerant: failures logged, not propagated.
3. Return nil even if some Stops failed (best-effort; aligned with F10.3 cleanup semantics).
```

### 6.5. `GetTranscript(ctx, sessionID)`

```
1. Load row.
2. If row.ClaudeUUID is nil → return [] (graceful).
3. worktree := store.WorktreesRepo.GetByID(row.WorktreeID)
4. path, err := sandbox.FindTranscriptFile(claudeHome, worktree.Path, *row.ClaudeUUID)
   - ErrTranscriptNotFound → return [] (graceful)
5. messages, err := sandbox.ParseTranscript(path)
6. Return messages.
```

`claudeHome` defaults to `~/.claude` (overridable via `JARVIS_CLAUDE_HOME` env for tests).

## 7. `TasksAPI` cleanup wiring (cross-cutting)

```go
type SessionCleanupFn func(ctx context.Context, taskID string) error

func NewTasksAPI(
    repo TasksRepo, bus events.Emitter,
    worktreeCleanup WorktreeCleanupFn,
    sessionCleanup SessionCleanupFn,   // NEW
) *TasksAPI

// Order in Patch (when state becomes terminal) AND in Discard:
//   1. emit task.updated / task.discarded
//   2. a.runSessionCleanup(ctx, id)     // NEW — stop subprocesses first
//   3. a.runWorktreeCleanup(ctx, id)    // F10.3 — remove FS after subprocess gone
```

**Why session cleanup before worktree:** if worktree disappears first, the subprocess of claude that's still writing to it becomes a zombie writing to a non-existent path. Stop first guarantees clean shutdown.

`runSessionCleanup`: same nil-tolerant pattern as `runWorktreeCleanup`. Errors logged via `log.Printf`, never propagated.

## 8. Event taxonomy

| Event | Trigger | Payload shape |
|---|---|---|
| `session.started` | `Start` ok | `*store.Session` |
| `session.identified` | First `Session ID: <uuid>` line from claude | `{id, task_id, claude_uuid}` |
| `session.status_changed` | `/hooks/Notification` or `/hooks/Stop` | `{id, task_id, previous, current}` |
| `session.tool_use` | `/hooks/PreToolUse` | `{id, task_id, tool}` |
| `session.stopped` | `Stop` ok or `/hooks/Stop` | `{id, task_id, exit_code}` |
| `session.exited` | Subprocess unexpectedly terminates | `{id, task_id, exit_code, stderr_tail}` |

`session.stopped` is fired for both graceful manual Stop AND the claude-hook-driven Stop event (semantically the session is idle/done; distinguishing is unnecessary for UI). `session.exited` is the abnormal/crash path.

## 9. Transcript parsing

### 9.1. File location

Claude writes to `~/.claude/projects/<encoded-cwd>/<session-uuid>.jsonl`, where `<encoded-cwd>` is the absolute cwd with `/` replaced by `-`. Example: cwd `/home/x/proj--feat` → `-home-x-proj--feat`.

```go
func FindTranscriptFile(claudeHome, worktreePath, claudeUUID string) (string, error) {
    encoded := strings.ReplaceAll(filepath.Clean(worktreePath), string(filepath.Separator), "-")
    path := filepath.Join(claudeHome, "projects", encoded, claudeUUID + ".jsonl")
    if _, err := os.Stat(path); err != nil {
        return "", ErrTranscriptNotFound
    }
    return path, nil
}
```

### 9.2. Parsing

Tolerant line-by-line scan:

```go
type rawEntry struct {
    Type      string          `json:"type"`
    Message   json.RawMessage `json:"message"`
    Content   json.RawMessage `json:"content"`
    Timestamp time.Time       `json:"timestamp"`
}

// Mapping rules:
//   type=user      → 1 TranscriptMessage{role=user, content=plain text}
//   type=assistant → fan-out: 1 per content block (text → role=assistant; tool_use → role=tool_use + ToolName)
//   type=tool_result → 1 message{role=tool_result, content=stringified}
//   unknown type → skip + log (non-fatal)
//   malformed JSON line → skip + log (non-fatal)
```

Schema drift mitigation: testdata fixtures captured from real claude (versioned in `internal/sandbox/testdata/transcripts/`). Parser failures on a fixture trigger investigation, not test breakage cascade.

### 9.3. `claudeUUID` discovery from PTY

Reader goroutine in §6.1 step 7 uses `regexp.MustCompile("Session ID: ([0-9a-f-]{36})")` against each line. First match captured and persisted. Failure mode (line never appears) leaves `claude_uuid` as `nil` indefinitely — Resume + GetTranscript both degrade to disabled / empty result.

- `internal/core/sessions_integration_test.go`: combines fake-aijail + real hook server + real DB. Validates settings.json content matches contract, subprocess receives the right `JARVIS_HOOK_URL`/`TOKEN`, simulated curl from fake-aijail triggers bus emit, Stop cleans everything. **This is the most valuable test in the suite — it asserts the contract that holds the subsystem together.**

### 11.3. E2E (Playwright)

`ui/e2e/sessions-flow.spec.ts` — 4 new tests:

1. Card shows enabled `▶ Iniciar sessão` when sandbox available; disabled when not (env `JARVIS_E2E_NO_SANDBOX=1`).
2. Click creates session; panel opens; chip flips from `executing` to whatever status is forced via simulate-hook.
3. Simulated hook (POST `/e2e/sessions/simulate-hook`) drives the UI live: chip + transcript update without reload.
4. Stop closes the session; chip becomes `done`; transcript remains visible; row moves to history group.

New endpoint in `cmd/jarvis-e2e-http`: `POST /e2e/sessions/simulate-hook` accepts `{session_id, event, payload}` and replays it against the actual hook server. Lets Playwright drive the full chain without needing claude+ai-jail real on CI.

Boot toggle: env `JARVIS_E2E_RUNTIME=fake` makes `cmd/jarvis-e2e-http` inject `fakeRuntime` instead of `AijailRuntime`. Required because we don't want CI to need `ai-jail` installed.

## 12. Risk register

| # | Risk | Mitigation |
|---|------|------------|
| 1 | Claude JSONL schema changes between versions | Tolerant parser (skip unknown fields & types); versioned fixtures in testdata; replay degrades to "no transcript" gracefully, never crashes. |
| 2 | Hook server port leaks to another local process | Bind `127.0.0.1` only + token-per-session (256 bits crypto/rand) + revoke on Stop. Accepted (same model as Python). |
| 3 | Race between subprocess exit and manual Stop | DB row status is source of truth; idempotent: Stop of already-terminal session is no-op. `bus.Emit("session.stopped")` fired at most once via atomic check-and-set in service. |
| 4 | Claude doesn't emit `Session ID: <uuid>` (version mismatch) | `claude_uuid` stays nil; Resume + GetTranscript disabled for that session; main flow keeps working; tooltip explains. |
| 5 | `ai-jail` crashes mid-spawn (settings written, subprocess crashes) | Full rollback: revoke token, remove settings, delete DB row. Worktree NOT rolled back (idempotent FS state across retries). |
| 6 | Session cleanup blocks worktree cleanup if subprocess hangs | Stop has 5s timeout + SIGKILL force. `CleanupForTask` iterates tolerant (one stuck session doesn't block others nor block subsequent worktree cleanup). |
| 7 | Hook server fails to bind (port unavailable) | Boot fail-fast with clear message. Status `health.sandbox_available=false` via same UI path. Button disabled. |
| 8 | Settings.json overwrites user's custom claude settings | The `.claude/` dir lives inside the worktree (per-task, isolated). Worktree is J-arvis-managed throwaway state. No user content collision. |
| 9 | Hook PreToolUse returns non-zero, claude blocks | `; exit 0` suffix in command guarantees we never block claude's tool execution regardless of curl/server state. |
| 10 | Token leak via logs (DB row mirror, error messages) | `HookToken` field has `json:"-"` tag; logging code reviewed to never log full `Session` struct unredacted. |

## 14. What this design does NOT include (out of scope, future)

- Multiple concurrent active sessions per task (UNIQUE constraint enforces 1; future feature could relax with a `branch` discriminator).
- Master session integration (Block C / F10.5).
- MCP tools driven by claude inside ai-jail (Block C / F10.5).
- Catalog / templates / permission profiles applied to spawn args (Block C / F10.5).
- Notifications/sink (libnotify) wired to status changes — Python has it; Go version deferred to F10.7 OS integration block.
- Run-from-panel Docker integration (Block D / F10.6).
- Pure-Go ai-jail port (ADR-0001 reaffirms external binary; never planned).
