# F10.8 spec #1 — Python Cleanup (design)

> **Status:** design (brainstorm 2026-05-18). Primeira de duas specs do bloco
> F10.8 (final do roadmap F10). Esta cobre: deletar todo o código Python do
> repo (daemon `orchestrator/`, testes `pytest`, `alembic/`, deps Python),
> renomear `Makefile.f10` → `Makefile`, e rewrite das docs `ARCHITECTURE.md`
> + `PRODUCT.md` pra refletir a realidade Go+Wails. Spec #2 (próxima) tratará
> packaging (.deb + .AppImage + assets desktop).

## Problema

O J-arvis começou como Python+FastAPI (até F9). A pivotação pra Go+Wails
(F10) substituiu tudo — daemon, store, sandbox runtime, API layer, hooks
— mas o código antigo permaneceu no repo. Resultado:

- Quem clona o repo hoje vê `orchestrator/`, `pyproject.toml`, `tests/`
  e razoavelmente pensa que é um projeto Python. `main.go` é o
  entrypoint real.
- `ARCHITECTURE.md` linha 25 ainda descreve "Orchestrator daemon (Python
  + FastAPI)". `PRODUCT.md` linha 30 lista "Daemon: Python 3.13 + FastAPI
  + SQLAlchemy". Docs vivos mentem.
- 2 Makefiles coexistem (`Makefile` Python-era + `Makefile.f10` Go).
  Onboarding de novo dev cai no Makefile errado.
- ~3000 LOC de Python pesando reviews + `git grep` ruidoso.

Spec #1 limpa tudo isso em 3 commits sequenciais (cada um deixa o repo
num estado consistente).

## Não-objetivos

- Packaging (.deb, .AppImage) — vai pro spec #2.
- CI/CD workflows — não existem em `.github/`, fora do MVP.
- Rewrite de specs/plans antigos em `docs/superpowers/` — registro
  histórico, devem permanecer.
- Rewrite de `CHANGELOG.md` — registro histórico legítimo.
- ADR novo documentando "removemos Python" — ADR-0007 (Go+Wails pivot)
  já contém o racional, YAGNI.
- Branch/tag arquivando o último estado Python — `git log --all
  --full-history -- orchestrator/` recupera quando precisar.

## Decisões de design (do brainstorm)

| Pergunta | Resposta |
|---|---|
| Estratégia do Makefile | **Renomear** `Makefile.f10 → Makefile`. Deleta o antigo (Python-era). User volta a usar `make start` em vez de `make -f Makefile.f10 start` |
| Doc updates (ARCHITECTURE/PRODUCT) | **Reescrever neste spec**. Não substitui por placeholder — escreve a doc Go+Wails de verdade |
| Preservação histórica do código Python | **Nada explícito** — git log basta. `git checkout <pre-cleanup-sha> -- orchestrator/` recupera quando precisar |
| `.orchestrator/data/` no .gitignore | **Remover** — era da era Python. Go app usa SQLite em `~/.local/share/jarvis/`, manifest em `.orchestrator/run.yml` (commitado, não dado) |

## Arquitetura: 3 commits sequenciais

Cada commit deixa o repo num estado consistente (build + tests verdes).

```
┌─ Commit 1: chore(F10.8.C.1): delete Python orchestrator ─────────────────┐
│ • git rm -r orchestrator/ tests/ alembic/                                │
│ • git rm alembic.ini pyproject.toml uv.lock                              │
│ • git rm Dockerfile.orchestrator .dockerignore                           │
│ • git rm scripts/f8-followups.sh scripts/f8-merge.sh scripts/f8-push.sh  │
│ • Modify .gitignore (remove 9 Python-only entries)                       │
│ • Verify: go vet + go build + go test all green                          │
└──────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─ Commit 2: chore(F10.8.C.2): rename Makefile.f10 → Makefile ─────────────┐
│ • git rm Makefile  (the old Python-era one)                              │
│ • git mv Makefile.f10 Makefile                                           │
│ • grep -r "Makefile.f10" --include='*.md' . in live docs (README,        │
│   CONTEXT.md, gotchas.md) → substitute by "Makefile". Historical specs/  │
│   plans NOT modified (record of when that was reality).                  │
│ • Verify: make start --dry-run works                                     │
└──────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─ Commit 3: docs(F10.8.C.3): rewrite ARCHITECTURE.md + PRODUCT.md ────────┐
│ • ARCHITECTURE.md: rewrite topology diagram, dir tree, test stack        │
│   section. Replace Python+FastAPI references with Go+Wails reality.      │
│ • PRODUCT.md: rewrite "Stack técnica resumida" section + scan remaining  │
│   11 mentions of Python/FastAPI/orchestrator+/uvicorn/alembic/sqlalchemy │
│   for contextual replacement.                                            │
│ • Verify: grep -ic "python\|fastapi" ARCHITECTURE.md PRODUCT.md ≈ 0      │
└──────────────────────────────────────────────────────────────────────────┘
```

Por que 3 commits separados (vs 1 big delete):

- Commit 1 é destrutivo e voluminoso (~3000 LOC removidas). Diff óbvio
  ("tudo daquele subdir foi").
- Commit 2 é renomeação pura — git rename detection preserva
  history; review é trivial.
- Commit 3 é doc prosa — revisão de prosa separada do code review.

```bash

go test -tags webkit2_41 -race ./internal/...
```

## Riscos

| Risco | Mitigação |
|---|---|
| Algum código Go ainda importa do `orchestrator/` via path shadow | `go vet` + `go build` na verificação pós-commit-1 pega. Improvável (F10.5/6/7 verificaram main.go é 100% Go) |
| `tests/` continha E2E validando o binário Go (não só FastAPI) | Pré-execução: `find tests/ -name '*.py' \| xargs grep -l 'build/bin\|jarvis\|Wails'`. Se algo: migrar pra Playwright em `ui/` antes de deletar. (Cobertura E2E hoje vive em `ui/e2e/` via Playwright contra `cmd/jarvis-e2e-http`.) |
| `Dockerfile.orchestrator` referenciado em algum lugar (CI, docs vivos) | `grep -r 'Dockerfile.orchestrator' . --exclude-dir=docs/superpowers --exclude-dir=docs/adr` deve retornar zero |
| `.dockerignore` é usado por outro Dockerfile remanescente | Não há outros Dockerfiles. Pode deletar com segurança |
| `Makefile.f10` mencionado em onboarding docs (README, etc) | grep + substituir por `Makefile`. Histórico em specs/plans/ADRs NÃO retrofittado |
| User roda `make -f Makefile.f10 ...` por hábito | Erro "No such file" óbvio. Rapidamente reaprende `make ...` |
| ARCHITECTURE.md rewrite descobre buracos ("como ai-jail interage com Wails?") | Trabalho contained (~50 linhas). Se aparecer buraco, parar e perguntar — não inventar |
| Specs/plans antigos em `docs/superpowers/` referenciam código que não existe mais | **Aceito como histórico.** Não retrofittar. ARCHITECTURE/PRODUCT (docs vivos) refletem realidade |
